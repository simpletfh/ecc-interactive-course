Spot the Bug: When Sequential Goes Wrong

You type /plan for a complex feature. What happens behind the scenes? Look at this code and find the problem.

What's wrong with this approach?

1 const agents = ['planner', 'architect', 'tdd-guide'];
2 for (const agent of agents) {
3 await runAgent(agent); // Waits for each to finish
4 }

Why This Matters

The planner needs to run FIRST to create a plan. But once the plan exists, architect and tdd-guide can work from it at the same time — they don't need to wait for each other!

Here's the better approach:

CODE 
// Run planner first to create the plan
const result = await runAgent('planner');

// Then run architect and tdd-guide in parallel
await Promise.all([
  runAgent('architect', result),
  runAgent('tdd-guide', result)
]);
PLAIN ENGLISH

Wait for the planner to finish and save its result...

Then launch both architect and tdd-guide at the same time...

Both get the same plan, and we wait for both to complete...

⚡
Speed Win!

Running architect and tdd-guide in parallel cuts the total time almost in half! Instead of waiting for one then the other, they work simultaneously — like having two chefs preparing different parts of a meal at the same time.

Smart Package Manager Detection

ECC automatically detects which package manager you're using — npm, pnpm, yarn, or bun. No manual configuration needed.

CODE 
export function detectPackageManager(cwd) {
  const lockFiles = [
    { file: 'pnpm-lock.yaml', manager: 'pnpm' },
    { file: 'yarn.lock', manager: 'yarn' },
    { file: 'package-lock.json', manager: 'npm' },
    { file: 'bun.lockb', manager: 'bun' }
  ];

  for (const { file, manager } of lockFiles) {
    if (existsSync(join(cwd, file))) {
      return manager;
    }
  }
  return 'npm'; // default fallback
}
PLAIN ENGLISH

This function figures out which package manager your project uses by looking for lock files in your project directory...

Check each known lock file type in order — if pnpm-lock.yaml exists, you're using pnpm...

If yarn.lock exists, you're using yarn...

If package-lock.json exists, you're using npm...

If bun.lockb exists, you're using bun...

Return the first match found. If no lock file exists, default to npm (the most common)...

💡
Why lock files matter

Lock files record the exact version of every dependency. They ensure your teammate's computer, the production server, and your CI environment all run the same code — no "works on my machine" problems.
