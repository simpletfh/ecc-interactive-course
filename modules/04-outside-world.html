Smart Package Manager Detection

ECC automatically detects which package manager you're using — npm, pnpm, yarn, or bun. No manual configuration needed.

CODE 
export function detectPackageManager(cwd) {
  const lockFiles = [
    { file: 'pnpm-lock.yaml', manager: 'pnpm' },
    { file: 'yarn.lock', manager: 'yarn' },
    { file: 'package-lock.json', manager: 'npm' },
    { file: 'bun.lockb', manager: 'bun' }
  ];

  for (const { file, manager } of lockFiles) {
    if (existsSync(join(cwd, file))) {
      return manager;
    }
  }
  return 'npm'; // default fallback
}
PLAIN ENGLISH

This function figures out which package manager your project uses by looking for lock files in your project directory...

Check each known lock file type in order — if pnpm-lock.yaml exists, you're using pnpm...

If yarn.lock exists, you're using yarn...

If package-lock.json exists, you're using npm...

If bun.lockb exists, you're using bun...

Return the first match found. If no lock file exists, default to npm (the most common)...

💡
Why lock files matter

Lock files record the exact version of every dependency. They ensure your teammate's computer, the production server, and your CI environment all run the same code — no "works on my machine" problems.
